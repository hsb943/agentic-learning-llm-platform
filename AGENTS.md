# Verdict

Give the direct answer in 1-3 lines.

# What Happened

Explain exactly what caused the issue.

# Why It Happened

Explain the underlying logic, flow, or bug.

# Fix

Provide the exact change needed.

# Code

Always provide the corrected code in a separate code block.

# Important Notes

Mention caveats, edge cases, notebook issues, environment issues, or common mistakes.

# Bottom Line

End with a one-paragraph summary.

Formatting rules:

* Use short sections with headings
* Use bullet points, numbered bullet points 
* Avoid long paragraphs
* Avoid mixing multiple ideas in one paragraph
* Use numbered steps when describing flow
* Put code in separate code blocks
* Separate diagnosis from solution
* Prefer scan-friendly formatting over prose
* Explain where the bug starts and where it ends
* If there are multiple causes, separate them into "Primary Cause" and "Secondary Cause"
